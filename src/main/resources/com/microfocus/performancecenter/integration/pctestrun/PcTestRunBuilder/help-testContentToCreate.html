The content of the field must be in yaml syntax (indentation and carriage return are important) composed according to the parameters described in the tables below.

Important: all parameters must be in lowercase.

test:
Parameter Description Required
name This will be the test name. Required
testfolderpath This will be the location of the test in the Test Management folder tree (e.g. "MyMainFolder\\MySubfolder\\MySubSubFolder"). Double backslash character should be used as folder separator. The Root folder ('Subject') should not be specified. Required
content The content of the test that requires additonal parameters specified in the table below. Required

content:
Parameter Description Required
controller Defines the controller to be used during the test execution (must be a Host available in the Performance Center project). If not specified, a controller will be chosen among the different controllers available in the Performance Center project. Not required.
lgamount Number of Load Generators to allocate to the test (all groups in the test will be executed by the same Load Generators). Not required if each group defined in the 'groups' parameter defines the LoadGenerator(s) it will be using via 'lgname' parameter (see 'groups' table below)
groups List in which every member defines a script (or group) to be included in the test. The parameter to be used in each member are specified in the 'groups' table below. Required
scheduler Can help define the duration of a test and if the Virtual Users should be started simultaneously or gradually. See table below. Not required

groups:
Parameter Description Required
groupname Name of the group (must be unique if several groups are defined). Required
vusers Number of Virtual Users to use for running the script (group). Required
scriptid Id of the script in the Performance Center project. Not required if 'scriptfullname' parameter is specified
scriptfullname Script folder and script name of the script (for example "MyMainFolder\\MySubFolder\\MyScriptName') to be added to the group. The root folder of Performance Center named "Subject" should not be specified and a double backslash should be used for separators). Not required if 'scriptid' parameter is specified
lgname List of Load Generators to be used to run the script (or group). The supported values are:
  • For using an existing LoadGenerator defined in Performance Center as host, specify its hostname as defined in Performance Center.
  • For using an automatically matched LoadGenerator, specify the value "LG" followed by a number (recommended).
  • For using a dynamic LoadGenerator (this is available if your Performance Center server and project are set to work with Docker), use the value "DOCKER" followed by a number.
 Not required and will be ignored if 'lgamount' parameter is specified.

scheduler:
Parameter Description Required
rampuptimeinseconds if specified, the Virtual Users will be started gradually (every 15 seconds, additional Virtual Users will be added to the execution of the test until the time specified in the parameter ends). Not required
durationinseconds Total time it will take to execute the test after all Virtual Users are started. Not required

In the example below:
  • The test name is "mytestname"
  • It will be placed under the folder path "Subject\\Tests\\mytests" (the root folder of the folders which is "Subject" should not be specified). The folder separator is a double backslash.
  • In the content:
    • We do not specify the controller to be used in the 'controller' parameter which means a random available controller will be used to execute the test.
    • We do not specify the 'lgamount' parameter which means we rely on the 'lgname' parameter specified in each group.
    • in the groups:
      • We add two scripts and for each, we provide a value in the 'groupname' parameter (the value should be unique for each group) and the number of Virtual Users used to run the group.
      • As we do not know the ID of the scripts, we use the 'scriptfullname' parameter in which we only mention the path of the script (without "Subject") followed by the script name. The folder separator is a double backslash.
      • We specify the Load Generators that will be used by each group (in this case, Load Generators will be automatically matched as we use the 'LG' prefix).
  • In the scheduler:
    • We want all Virtual Users to be initialized gradually (45 seconds).
    • We want the test to stop after 5 minutes (300 seconds).

##################################################
#test
name: mytestname
testfolderpath: "Tests\\mytests"
content:
#  controller: "mycontroller"
#  lgamount: '2'
  groups:
    - groupname: "TEstInt"
      vusers: '20'
#      scriptid: '1'
      scriptfullname: "plugin\\TEstInt"
      lgname:
        - "LG1"
        - "LG2"
    - groupname: "Mtours"
      vusers: '20'
#      scriptid: '2'
      scriptfullname: "plugin\\mtours"
      lgname:
        - "LG3"
        - "LG4"
  scheduler:
    rampuptimeinseconds: '45'
    durationinseconds: '300'
##################################################
